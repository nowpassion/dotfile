## Basic Setup

Minimal configuration in `init.lua`:

```lua
-- Add to your init.lua
require('ncpf.ncpf').init()
```

## With Lazy.nvim

Complete lazy.nvim setup with all recommended plugins:

```lua
return {
  -- ncpf.nvim - C/C++ project management
  {
    'nowpassion/ncpf.nvim',
    ft = { 'c', 'cpp', 'h', 'hpp', 'cc' },
    dependencies = {
      'nvim-telescope/telescope.nvim',
      'neovim/nvim-lspconfig',
      'hrsh7th/nvim-cmp',
      'hrsh7th/cmp-nvim-lsp',
    },
    config = function()
      -- Default: search from home directory
      require('ncpf.ncpf').init()
      
      -- Or specify custom top directory
      -- require('ncpf.ncpf').init('/home/user/projects')
    end,
  },
  
  -- Required plugins for full functionality
  {
    'nvim-telescope/telescope.nvim',
    dependencies = { 'nvim-lua/plenary.nvim' },
  },
  
  {
    'neovim/nvim-lspconfig',
  },
  
  {
    'hrsh7th/nvim-cmp',
    dependencies = {
      'hrsh7th/cmp-nvim-lsp',
      'hrsh7th/cmp-buffer',
      'hrsh7th/cmp-path',
    },
  },
}
```

## With Packer.nvim

Complete packer.nvim setup:

```lua
return require('packer').startup(function(use)
  -- ncpf.nvim
  use {
    'nowpassion/ncpf.nvim',
    ft = { 'c', 'cpp', 'h', 'hpp', 'cc' },
    config = function()
      require('ncpf.ncpf').init()
    end,
  }
  
  -- Required plugins
  use {
    'nvim-telescope/telescope.nvim',
    requires = { 'nvim-lua/plenary.nvim' }
  }
  
  use 'neovim/nvim-lspconfig'
  
  use {
    'hrsh7th/nvim-cmp',
    requires = {
      'hrsh7th/cmp-nvim-lsp',
      'hrsh7th/cmp-buffer',
      'hrsh7th/cmp-path',
    }
  }
end)
```

## Project Examples

### Example 1: Simple CMake Project

```
my-project/
├── .nvim_c_project_root          # Project marker
├── CMakeLists.txt
├── compile_commands.json         # Generated by CMake
├── src/
│   ├── main.c
│   └── utils.c
└── include/
    └── utils.h
```

Setup:
```bash
cd my-project
touch .nvim_c_project_root
cmake -DCMAKE_EXPORT_COMPILE_COMMANDS=1 .
```

### Example 2: Makefile Project with Bear

```
legacy-project/
├── .nvim_c_project_root          # Project marker
├── Makefile
├── compile_commands.json         # Generated by Bear
└── src/
    ├── main.c
    └── module.c
```

Setup:
```bash
cd legacy-project
touch .nvim_c_project_root
bear -- make
```

### Example 3: Multi-directory Build

```
complex-project/
├── .nvim_c_project_root          # Project marker
├── .ncpf_clangd_option          # Custom clangd config
├── build/
│   └── compile_commands.json
└── src/
    └── main.cpp
```

`.ncpf_clangd_option` content:
```
--compile-commands-dir=/home/user/complex-project/build
```

### Example 4: Cross-compilation Project

```
embedded-project/
├── .nvim_c_project_root
├── .ncpf_clangd_option
├── compile_commands.json
└── src/
    └── firmware.c
```

`.ncpf_clangd_option` content:
```
--compile-commands-dir=/home/user/embedded-project
--query-driver=/opt/arm-toolchain/bin/arm-none-eabi-gcc
```

### Example 5: Using cscope

```
cscope-project/
├── .nvim_c_project_root
├── cscope.out                    # Generated by cscope
└── src/
    └── main.c
```

Setup:
```bash
cd cscope-project
touch .nvim_c_project_root
cscope -Rb  # Generate cscope database
```

### Example 6: Using ctags

```
ctags-project/
├── .nvim_c_project_root
├── tags                          # Generated by ctags
└── src/
    └── main.c
```

Setup:
```bash
cd ctags-project
touch .nvim_c_project_root
ctags -R .  # Generate tags file
```

## Custom Top Directory

Limit project search to a specific directory:

```lua
-- Only search for projects under /home/user/dev
require('ncpf.ncpf').init('/home/user/dev')
```

This is useful when:
- You want to restrict search scope
- You have multiple development directories
- You want to avoid searching entire home directory

## Tips

1. **Performance**: Place `.nvim_c_project_root` at the actual project root to avoid unnecessary upward directory searches

2. **Multiple Projects**: Each project should have its own `.nvim_c_project_root` marker

3. **Priority**: ncpf checks tools in this order:
   - clangd (best for modern C/C++)
   - coc.nvim + ccls
   - cscope
   - ctags

4. **Regeneration**: Remember to regenerate compile_commands.json, cscope.out, or tags when:
   - Adding new files
   - Changing build configuration
   - Modifying project structure
